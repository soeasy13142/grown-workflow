# 命令：`/e2e` — 端到端测试

> 本命令基于 ECC（affaan-m/ECC, MIT）的 `/e2e`。详见 [CREDITS.md](../../../CREDITS.md)。

## 何时用

- 完整的用户流程验证
- 跨模块/跨服务的集成场景
- 发布前的回归检查

## 本仓库约定

1. **先单元测试（/tdd），再 e2e** — e2e 覆盖关键用户路径，不试图覆盖所有分支
2. **e2e 应可重复运行** — 不依赖外部状态（数据库、第三方 API）
3. **失败时提供足够上下文** — 截图、日志、请求/响应快照

## 用法

```
在对话中直接输入：
/e2e

然后描述要测试的端到端场景。
```

ECC 会分析代码并生成 e2e 测试用例。

## 配合使用

| 阶段 | 命令 |
|------|------|
| 单元测试 | `/tdd` |
| 覆盖率分析 | `/ecc:test-coverage` |
| 发布前全量检查 | `/ecc:code-review` |
