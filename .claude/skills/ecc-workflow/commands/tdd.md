# 命令：`/tdd` — TDD 工作流

> 本命令基于 ECC（affaan-m/ECC, MIT）的 `/tdd`。详见 [CREDITS.md](../../../CREDITS.md)。

## 何时用

新功能开发、有明确输入/输出边界的模块。

## 本仓库约定

1. **先写测试** — 新功能必须先从测试用例开始
2. **目标覆盖率 ≥ 80%** — 见 `.claude/rules/common/testing.md`
3. **测试不是凑数** — 不要仅为通过覆盖率而写无效测试
4. **ECC 的 /tdd 流程** — 自动引导：写测试 → 跑 → 红 → 写实现 → 绿 → 重构

## 用法

```
在对话中直接输入：
/tdd

然后描述你要实现的功能。
```

ECC 会引导你完成红-绿-重构循环。

## 注意

- /tdd 不适用于：纯探索性任务、架构设计、数据迁移
- 如果功能实现后覆盖率不达标，走 `/ecc:test-coverage`
